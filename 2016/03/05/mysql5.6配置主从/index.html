Mysql5.6配置主从

Posted on 2016-03-05 | In Mysql

修改mysql配置文件，增加同步参数：
mkdir /mysqllog
主：
server-id=1

BINARY LOGGING

innodb_log_group_home_dir = /mysqllog/
log-bin = /mysqllog/master-bin
expire-logs-days = 7
sync_binlog=1
binlog-ignore-db = mysql
binlog-ignore-db = performance_schema
binlog-ignore-db = information_schema

从：
server-id=2

BINARY LOGGING

innodb_log_group_home_dir = /mysqllog/
log-bin = /mysqllog/slave-bin
expire-logs-days = 7
sync_binlog=1
binlog-ignore-db = mysql
binlog-ignore-db = performance_schema
binlog-ignore-db = information_schema
log_bin_trust_function_creators=TRUE
slave-skip-errors = all #跳过主从复制出现的错误

重启两个数据库

主库新建账号并授权slave
grant replication slave on _._ to ‘sync‘@’%’ identified by ‘lhtx_123456’;
flush privileges;

检测授权结果:
select * from user where host=’%’and user=’sync’\G;
如果Repl_slave_priv项为Y，表示授权成功。

登录主数据库查看master状态
show master status;

注：执行完此步骤后不要再操作主服务器MYSQL，防止主服务器状态值变化

配置从数据库slave
show master status;

找出“File：”，“Position：”

CHANGE MASTER TO MASTER_HOST=’15.3.2.12’, MASTER_USER=’sync’,MASTER_PASSWORD=’lhtx_123456’,MASTER_LOG_FILE=’#######’,MASTER_LOG_POS=########;
注：如果slave处于启动状态，则先执行stopslave命令停止，再执行changemaster to…配置slave，最后执行start slave启动slave。

检测从数据库复制功能状态
show slave status\G;
其中下面两项都必须为yes。

Slave_IO_Running: Yes

Slave_SQL_Running: Yes
