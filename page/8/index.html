Centos网卡做bond0

Posted on 2016-03-05 | In Centos

mode=0：平衡负载模式，有自动备援，但需要”Switch”支援及设定。

mode=1：自动备援模式，其中一条线若断线，其他线路将会自动备援。

系统启动时加载bonding模块：
添加一个bind配置文件：
vi /etc/modprobe.d/bonding.conf
alias bond0 bonding

网卡配置文件：

ifcfg-bond0：

DEVICE=bond0
BOOTPROTO=none
ONBOOT=yes
IPADDR=10.3.8.4
NETMASK=255.255.0.0
GATEWAY=10.3.1.254
USERCTL=no
BONDING_OPTS=”mode=0 miimon=100”

ifcfg-em1：

DEVICE=em1
MASTER=bond0
SLAVE=yes
USERCTL=no
ONBOOT=yes
BOOTPROTO=none

ifcfg-em2：

DEVICE=em2
MASTER=bond0
SLAVE=yes
USERCTL=no
ONBOOT=yes
BOOTPROTO=none

Zabbix监控mysql

Posted on 2016-03-05 | In Zabbix

在mysql中建立zabbix用户
GRANTUSAGEON_._TO‘zabbix‘@’localhost’IDENTIFIED BY’*‘;
FLUSHPRIVILEGES;

/etc/zabbix/.my.cnf以提供Zabbix Agent访问数据库，内容类似如下：

Zabbix Agent

[mysql]
host = localhost
user = zabbix
password = ***

#（要修改）
socket = /var/run/mysqld/mysqld.sock
[mysqladmin]
host = localhost
user = zabbix
password = **

#（要修改）
socket = /var/run/mysqld/mysqld.sock

保存此文件然后修改/etc/zabbix/zabbix_agentd.d/userparameter_mysql.conf，这个文件用于制定Zabbix Agent如何获取MySQL数据，我们需要将所有的/var/lib/zabbix修改为当前.my.cnf文件的路径/etc/zabbix。设置完成之后保存并重启

zabbix只对/usr/bin和/usr/sbin下的环境变量识别，所以如果编译安装的mysql就要在/usr/bin/下建立软连接
ln -s /data/mysql/bin/mysql /usr/bin/mysql
ln -s /data/mysql/bin/mysqladmin /usr/bin/mysqladmin

参考链接：http://www.kaijia.me/2014/01/use-zabbix-monitor-mysql-server/

Centos6.5部署zabbix2.4.7过程

Posted on 2016-03-05 | In Zabbix

首先配置zabbix服务器能通外网
关闭iptables和selinux
安装zabbix存储库包
rpm -ivh http://repo.zabbix.com/zabbix/2.4/rhel/6/x86_64/zabbix-release-2.4-1.el6.noarch.rpm
安装zabbix
yum install zabbix-server-mysql zabbix-web-mysql
安装mysql
yum install mysql-server -y
安装ntp
yum install ntp -y
ntpdate time.nist.gov
设置开机启动
chkconfig ntpd on
chkconfig httpd on
chkconfig mysqld on
chkconfig zabbix-server on
启动服务
/etc/init.d/ntpd start
/etc/init.d/httpd start
/etc/init.d/mysqld start
/etc/init.d/zabbix-server start
创建zabbix数据库
mysql -uroot -p
create database zabbix character set utf8 collate utf8_bin;
设置mysql访问权限，只允许本机访问
grant all privileges on zabbix.* to zabbix@localhost identified by ‘zabbix’;
flush privileges;
quit；
导入数据表结构
cd /usr/share/doc/zabbix-server-mysql-2.4.7/create
mysql -uzabbix -pzabbix zabbix < schema.sql
mysql -uzabbix -pzabbix zabbix < images.sql
mysql -uzabbix -pzabbix zabbix < data.sql
编辑zabbix配置文件指向数据库
vi /etc/zabbix/zabbix_server.conf
DBHost=localhost
DBName=zabbix
DBUser=zabbix
DBPassword=zabbix

修改httpd下的zabbix配置文件
vi /etc/httpd/conf.d/zabbix.conf
php_value max_execution_time 300
php_value memory_limit 128M
php_value post_max_size 16M
php_value upload_max_filesize 2M
php_value max_input_time 300
php_value date.timezone Asia/Shanghai
service httpd restart

添加iptables规则
-A INPUT -p tcp -m tcp –dport 10050 -j ACCEPT
-A INPUT -p udp -m udp –dport 10050 -j ACCEPT
-A INPUT -p tcp -m tcp –dport 10051 -j ACCEPT
-A INPUT -p udp -m udp –dport 10051 -j ACCEPT

进入web安装向导
http://服务器IP/zabbix

完成

完成后登陆
登陆链接：http://192.168.10.222/zabbix/index.php
默认账号Admin，密码zabbix

修改中文语言

Mysql-5.7.10安装配置&主备

Posted on 2016-03-05 | In Mysql

官网下载：http://cdn.mysql.com//Downloads/MySQL-5.7/mysql-5.7.10.tar.gz
上传到/root/backup/下解压
准备工作：
安装编译依赖包：yum -y install make gcc-c++ cmake bison-devel ncurses-devel
防火墙开3306端口：

检查当前系统有没有旧版本mysql，有就强制删：

从MySQL 5.7.5开始Boost库是必需的，下载Boost库，最新稳定版：
http://nchc.dl.sourceforge.net/project/boost/boost/1.59.0/boost_1_59_0.tar.gz
上传到/root/backup/下解压并进入目录：
给权限： chmod -R 755 boost_1_59_0
编译并安装：
./bootstrap.sh

./b2

./b2 install

编译安装：
给权限：chmod -R 755 mysql-5.6.28，进入目录
编译：
cmake -DCMAKE_INSTALL_PREFIX=/data/mysql -DMYSQL_UNIX_ADDR=/data/mysql/mysql.sock -DMYSQL_DATADIR=/data/mysql/data -DDEFAULT_CHARSET=utf8 -DDEFAULT_COLLATION=utf8_general_ci -DWITH_EXTRA_CHARSETS:STRING=utf8,gbk -DWITH_MYISAM_STORAGE_ENGINE=1 -DWITH_INNOBASE_STORAGE_ENGINE=1 -DWITH_MEMORY_STORAGE_ENGINE=1 -DENABLED_LOCAL_INFILE=1 -DMYSQL_USER=mysql -DMYSQL_TCP_PORT=3306

####################################################
　　cmake参数说明：
　　-DCMAKE_INSTALL_PREFIX= 数据库程序安装路径;
　　-DMYSQL_DATADIR= 数据库文件存放路径（不配置的话会默认创建$PREFIX_DIR/data）
　　-DMYSQL_UNIX_ADDR= 默认位置是/var/lib/mysql/mysql.sock，如果指定其他路径，需要做一个软链接，或者在配置文件my.cnf中进行设置。
　　-DDEFAULT_CHARSET= 默认数据库编码
　　-DDEFAULT_COLLATION=默认数据库整理编码
　　-DWITH_EXTRA_CHARSETS= 扩展支持编码(all | utf8,gbk,gb2312 | none)
　　-DWITH_MYISAM_STORAGE_ENGINE= MYISAM引擎支持(1|0)
　　-DWITH_INNOBASE_STORAGE_ENGINE= innoDB引擎支持(1|0)
　　-DWITH_MEMORY_STORAGE_ENGINE= MEMORY引擎支持(1|0)
-DENABLED_LOCAL_INFILE=指定是否允许本地执行LOAD DATA INFILE，默认OFF

####################################################

安装：
make -j grep processor /proc/cpuinfo | wc -l #编译很消耗系统资源，小内存可能编译通不过

make install

注意：若第一次的配置错误，需要先删除当前mysql源码目录中的CMakeCache.txt，然后再重新进行cmake配置。

配置PATH：
vi /etc/profile
末尾添加：
PATH=/data/mysql/bin:$PATH
export PATH

保存使其立即生效：
source /etc/profile

配置mysql：
做成系统服务：
cp /data/mysql/support-files/mysql.server /etc/init.d/mysqld
chkconfig mysqld on

新建mysql用户和用户组
groupadd mysql
useradd -g mysql mysql
修改mysql安装目录权限为mysql:mysql
chown -R mysql:mysql /data/mysql

创建mysql配置文件：
mv /etc/my.cnf /etc/my.cnf.bak
cp /data/mysql/support-files/my-default.cnf /etc/my.cnf

先修改一些基本参数，后续在做调优：

mysql初始化：
/data/mysql/bin/mysqld –initialize-insecure –user=mysql –basedir=/data/mysql –datadir=/data/mysql/data

#######################################
–initialize-insecure：不生成随机密码

#######################################

启动mysql：
/etc/init.d/mysqld start

注意：第一次启动会自动生成一个pid，把生成的pid路径写到my.cnf里

进入mysql设置密码：
mysql -uroot
SET PASSWORD=PASSWORD(‘‘);
设置允许root用户远程访问：
GRANT ALL PRIVILEGES ON _._ TO ‘root‘@’%’ IDENTIFIED BY ‘*‘ WITH GRANT OPTION;
立即生效：
FLUSH PRIVILEGES;
退出：
quit

后期调优：
新建两个log文件夹：
mkdir /data/mysql/log
mkdir /data/mysql/binlog
my.cnf配置文件里的日志输出到这两个文件夹下

配置mysql主从：
在两台上都配置好binlog，
然后在db-master上运行“SHOW MASTER STATUS”即可看到master有状态

在master新建一个专门同步的用户：

记录下master上执行“SHOW MASTER STATUS”的值
然后在db-slave上运行change master
“CHANGE MASTER TO MASTER_HOST=’*‘, MASTER_USER=’sync’,MASTER_PASSWORD=’*‘,MASTER_LOG_FILE=’log.000001’,MASTER_LOG_POS=1289156;”

然后再db-salve上运行“START SLAVE ”，启动slave

然后再db-salve上运行“SHOW SLAVE STATUS”查看slave状态：

这样就主从做完了
