Mysql-5.7.10安装配置&主备

Posted on 2016-03-05 | In Mysql

官网下载：http://cdn.mysql.com//Downloads/MySQL-5.7/mysql-5.7.10.tar.gz
上传到/root/backup/下解压
准备工作：
安装编译依赖包：yum -y install make gcc-c++ cmake bison-devel ncurses-devel
防火墙开3306端口：

检查当前系统有没有旧版本mysql，有就强制删：

从MySQL 5.7.5开始Boost库是必需的，下载Boost库，最新稳定版：
http://nchc.dl.sourceforge.net/project/boost/boost/1.59.0/boost_1_59_0.tar.gz
上传到/root/backup/下解压并进入目录：
给权限： chmod -R 755 boost_1_59_0
编译并安装：
./bootstrap.sh

./b2

./b2 install

编译安装：
给权限：chmod -R 755 mysql-5.6.28，进入目录
编译：
cmake -DCMAKE_INSTALL_PREFIX=/data/mysql -DMYSQL_UNIX_ADDR=/data/mysql/mysql.sock -DMYSQL_DATADIR=/data/mysql/data -DDEFAULT_CHARSET=utf8 -DDEFAULT_COLLATION=utf8_general_ci -DWITH_EXTRA_CHARSETS:STRING=utf8,gbk -DWITH_MYISAM_STORAGE_ENGINE=1 -DWITH_INNOBASE_STORAGE_ENGINE=1 -DWITH_MEMORY_STORAGE_ENGINE=1 -DENABLED_LOCAL_INFILE=1 -DMYSQL_USER=mysql -DMYSQL_TCP_PORT=3306

####################################################
　　cmake参数说明：
　　-DCMAKE_INSTALL_PREFIX= 数据库程序安装路径;
　　-DMYSQL_DATADIR= 数据库文件存放路径（不配置的话会默认创建$PREFIX_DIR/data）
　　-DMYSQL_UNIX_ADDR= 默认位置是/var/lib/mysql/mysql.sock，如果指定其他路径，需要做一个软链接，或者在配置文件my.cnf中进行设置。
　　-DDEFAULT_CHARSET= 默认数据库编码
　　-DDEFAULT_COLLATION=默认数据库整理编码
　　-DWITH_EXTRA_CHARSETS= 扩展支持编码(all | utf8,gbk,gb2312 | none)
　　-DWITH_MYISAM_STORAGE_ENGINE= MYISAM引擎支持(1|0)
　　-DWITH_INNOBASE_STORAGE_ENGINE= innoDB引擎支持(1|0)
　　-DWITH_MEMORY_STORAGE_ENGINE= MEMORY引擎支持(1|0)
-DENABLED_LOCAL_INFILE=指定是否允许本地执行LOAD DATA INFILE，默认OFF

####################################################

安装：
make -j grep processor /proc/cpuinfo | wc -l #编译很消耗系统资源，小内存可能编译通不过

make install

注意：若第一次的配置错误，需要先删除当前mysql源码目录中的CMakeCache.txt，然后再重新进行cmake配置。

配置PATH：
vi /etc/profile
末尾添加：
PATH=/data/mysql/bin:$PATH
export PATH

保存使其立即生效：
source /etc/profile

配置mysql：
做成系统服务：
cp /data/mysql/support-files/mysql.server /etc/init.d/mysqld
chkconfig mysqld on

新建mysql用户和用户组
groupadd mysql
useradd -g mysql mysql
修改mysql安装目录权限为mysql:mysql
chown -R mysql:mysql /data/mysql

创建mysql配置文件：
mv /etc/my.cnf /etc/my.cnf.bak
cp /data/mysql/support-files/my-default.cnf /etc/my.cnf

先修改一些基本参数，后续在做调优：

mysql初始化：
/data/mysql/bin/mysqld –initialize-insecure –user=mysql –basedir=/data/mysql –datadir=/data/mysql/data

#######################################
–initialize-insecure：不生成随机密码

#######################################

启动mysql：
/etc/init.d/mysqld start

注意：第一次启动会自动生成一个pid，把生成的pid路径写到my.cnf里

进入mysql设置密码：
mysql -uroot
SET PASSWORD=PASSWORD(‘‘);
设置允许root用户远程访问：
GRANT ALL PRIVILEGES ON _._ TO ‘root‘@’%’ IDENTIFIED BY ‘*‘ WITH GRANT OPTION;
立即生效：
FLUSH PRIVILEGES;
退出：
quit

后期调优：
新建两个log文件夹：
mkdir /data/mysql/log
mkdir /data/mysql/binlog
my.cnf配置文件里的日志输出到这两个文件夹下

配置mysql主从：
在两台上都配置好binlog，
然后在db-master上运行“SHOW MASTER STATUS”即可看到master有状态

在master新建一个专门同步的用户：

记录下master上执行“SHOW MASTER STATUS”的值
然后在db-slave上运行change master
“CHANGE MASTER TO MASTER_HOST=’*‘, MASTER_USER=’sync’,MASTER_PASSWORD=’*‘,MASTER_LOG_FILE=’log.000001’,MASTER_LOG_POS=1289156;”

然后再db-salve上运行“START SLAVE ”，启动slave

然后再db-salve上运行“SHOW SLAVE STATUS”查看slave状态：

这样就主从做完了

db_online_host=”***“

同步库数据

######################

echo “DB czonline Sync Start… “date +'%Y-%m-%d %H:%M:%S'

for db in **
do
sql_file=”/data/$db.sql”
echo “$db Export Start…$sql_file “date +'%Y-%m-%d %H:%M:%S'
/usr/bin/mysqldump -h $db_host -u $db_username -p”$db_password” –set-g
tid-purged=OFF $db > $sql_file
echo “$dbImport Start… “date +'%Y-%m-%d %H:%M:%S'

sql_file

CentOS 6.4下ELK套件搭建日志分析和监控平台

Posted on 2016-03-05 | In ELK

简介：
ELK套件（ELK stack）是指ElasticSearch、Logstash和Kibana三件套。这三个软件可以组成一套日志分析和监控工具。
由于三个软件各自的版本号太多，建议采用ElasticSearch官网推荐的搭配组合：http://www.elasticsearch.org/overview/elkdownloads/
ELK官网：https://www.elastic.co/
参考链接：
http://blog.kazaff.me/2015/06/05/%E6%97%A5%E5%BF%97%E6%94%B6%E9%9B%86%E6%9E%B6%E6%9E%84--ELK/
这次使用版本：
ElasticSearch-2.1.1
Logstash-2.1.0
Kibana-4.3.1

开启iptables允许

vim /etc/sysconfig/iptables

-A INPUT -m state –state NEW -m tcp -p tcp –dport 80 -j ACCEPT

-A INPUT -m state –state NEW -m tcp -p tcp –dport 9200 -j ACCEPT

-A INPUT -m state –state NEW -m tcp -p tcp –dport 9292 -j ACCEPT

-A INPUT -m state –state NEW -m tcp -p tcp –dport 5601 -j ACCEPT

service iptables restart

安装JDK
yum -y install java-1.7.0-openjdk*

安装Logstash
（Logstash默认的对外服务的端口是9292。）
下载地址：https://download.elastic.co/logstash/logstash/logstash-all-plugins-2.1.0.tar.gz

上传到/root/back/下：
解压：
tar zxvf logstash-all-plugins-2.1.0.tar.gz
拷贝到/usr/local/下：
cp -r logstash-2.1.0 /usr/local/

/usr/local/logstash-2.1.0/下创建了两个文件夹，分别用来放配置文件和日志
mkdir conf logs

测试logstash：
./bin/logstash -e ‘input { stdin { } } output { stdout {} }’

安装ElasticSearch
（ElasticSearch默认的对外服务的HTTP端口是9200，节点间交互的TCP端口是9300。）
下载链接：https://download.elasticsearch.org/elasticsearch/release/org/elasticsearch/distribution/tar/elasticsearch/2.1.1/elasticsearch-2.1.1.tar.gz
上传到/root/backup/下：
解压：
tar zxvf elasticsearch-2.1.1.tar.gz
拷贝到/usr/local/下：
cp -r elasticsearch-2.1.1 /usr/local/

创建elasticsearch运行时所需要的系统账号（elasticsearch-2.0.0版本禁止使用root账号直接运行，所以需要创建账号用于运行elasticsearch）
useradd elasticsearch
切换到elasticsearch用户启动
su - elasticsearch
启动：
./bin/elasticsearch -d

安装kibana
下载地址：https://download.elastic.co/kibana/kibana/kibana-4.3.1-linux-x64.tar.gz
上传到/root/backup/下：
解压：
tar zxvf kibana-4.3.1-linux-x64.tar.gz
拷贝到/usr/local/下
cp -r kibana-4.3.1-linux-x64 /usr/local/
启动：
./bin/kibana

使用的是kibana默认的配置，现在可以直接在浏览器中访问： http://127.0.0.1:5601

cp -r kibana-4.3.1-linux-x64 /etc/kibana4.3.1

yum install python-setuptools

easy_install supervisor
生成配置文件：

echo_supervisord_conf >> /etc/supervisord.conf
编辑配置文件vi /etc/supervisord.conf
最后添加：
[include]
files = /etc/supervisord.d/*.conf

添加kibana配置文件：
mkdir /etc/supervisord.d
vi kibana.conf
添加：
[program:kibana]
command=/etc/kibana4.3.1/bin/kibana
autorestart = true
redirect = true
stderr_logfile = /var/log/kibana/kibana.log
startsecs = 3

给日志文件建目录：
mkdir /var/log/kibana
touch /var/log/kibana/kibana.log
启动：

/usr/bin/python /usr/bin/supervisord -c /etc/supervisord.conf
